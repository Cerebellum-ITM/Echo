## Architecture Decisions

- **connect impersonation = forge a session, never a password.** El mint
  reproduce `Session.finalize` (`root.session_store.new()` +
  `_compute_session_token`), no autentica con credenciales. 2FA queda
  bypasseado a propósito (uso dev); el único freno es el prod-confirm.
- **Cookie por CDP, no por bookmarklet ni helper-server.** La `session_id`
  de Odoo es HttpOnly: JS no puede sobreescribirla y un helper-server
  exigía abrir un puerto inbound (descartado). Chrome DevTools Protocol
  (`Network.setCookie`) la setea a nivel de red → cero puertos, cero
  módulo en Odoo.
- **Echo (para connect) corre en el laptop; el server solo se toca por
  SSH para mintear.** El navegador siempre es local, así que la cookie se
  aterriza local; lo único remoto es ejecutar el script Python en el
  contenedor.
- **El remoto hereda el perfil de Echo del server, no se re-declara en
  local.** `fetchRemoteProfile` lee por SSH `global.toml` +
  `projects/<sha256(remote_path)>.toml` (mismo hash que usa Echo local).
  Requirió persistir `project_path` en el perfil + auto-migración en el
  arranque (`BackfillProjectPath`).
- **Targets directos basados en `~/.ssh/config`.** Echo no gestiona SSH:
  referencia el alias del Host; el modo directo `echo connect <name>`
  guarda targets nombrados en `global.toml` y descubre proyectos leyendo
  la config de Echo del server (nunca escaneando dockers).
- **La versión del binario siempre muestra el commit.** `+<shortsha>`
  siempre, `.dirty` cuando el árbol está sucio (ldflags del Makefile), para
  poder identificar exactamente qué build corre — no solo cuando hay
  cambios sin commitear.
